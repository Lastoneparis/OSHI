# OSHI Messenger - Encryption Documentation

## Overview

OSHI uses state-of-the-art cryptographic algorithms to ensure that your messages and calls remain private. This document explains the encryption mechanisms in detail.

### Decryption Process

```
1. Parse header
2. Verify sender public key
3. Check message counter (replay protection)
4. Perform DH ratchet if needed
5. Derive message key from chain
6. Decrypt with AES-256-GCM
7. Verify authentication tag
8. Return plaintext or reject
```
